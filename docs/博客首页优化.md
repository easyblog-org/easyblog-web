# AI Coding 提示词：博客首页视觉重构（深色高级风格，Logo 使用“EasyBlog”文字）

> 目标：基于现有 Nuxt 2 + Tailwind CSS 博客项目，将首页改造为与个人作品集一致的深色高级风格。Logo 使用纯文本“EasyBlog”，导航栏、Hero、文章列表、页脚全部重新设计，移除原有侧边栏，采用宽屏网格布局。

## 一、全局样式重置（深色主题 + 宽屏）

1. 背景：径向渐变 `radial-gradient(circle at 10% 20%, #0a0f1c, #030614)`，全屏固定。
2. 文字颜色：默认 `#e2e8f0`，标题 `#ffffff`，链接 hover 变为 `#3b82f6`。
3. 最大内容宽度：`1280px`，左右 margin 自动，内边距移动端 `1rem`，桌面端 `2rem`。
4. 字体：`'Inter', 'PingFang SC', 'Microsoft YaHei', sans-serif`。标题加粗，行高 `1.2`。
5. 定义全局 CSS 变量或 Tailwind 主题扩展：主色 `#3b82f6`，卡片背景 `rgba(15, 23, 42, 0.6)`。

## 二、导航栏

- 固定顶部，初始透明，滚动后背景 `rgba(3, 6, 20, 0.8)` + `backdrop-filter: blur(12px)`。
- Logo：`<span class="text-xl font-bold text-white tracking-tight">EasyBlog</span>`，无额外装饰。
- 菜单项：首页、归档、分类、标签、关于（根据实际路由调整）。
- 移动端：汉堡菜单（☰），点击展开垂直菜单。
- 添加一个外部链接按钮“作品集”，样式为边框按钮（`border border-primary text-primary`）。

## 三、Hero 区块

在文章列表上方添加 Hero 区域：
- 大标题：`EasyBlog`（或 `技术笔记`，字号 56px 粗体）
- 副标题：`记录后端与AI的思考，分享可靠的技术实践`（字号 20px，浅灰色）
- 可选：搜索框（放大镜图标 + 输入框）或 RSS 链接。
- Hero 下方加一条细分割线或蓝色光晕。

## 四、文章列表（无侧边栏，宽屏网格）

1. 移除原有侧边栏，文章区域占满宽度。
2. 布局：响应式网格 `grid`，间隙 `1.5rem`。
   - 桌面：3 列
   - 平板：2 列
   - 手机：1 列
3. 卡片样式：
   - 背景 `rgba(15, 23, 42, 0.6)`，`backdrop-filter: blur(4px)`
   - 圆角 `1rem`
   - 内边距 `1.5rem`
   - 边框 `1px solid rgba(59,130,246,0.2)`
   - hover：边框变 `#3b82f6`，阴影 `0 20px 25px -12px rgba(0,0,0,0.5)`
4. 卡片内容：
   - 标题：`text-xl font-bold text-white mb-2`
   - 摘要：`text-sm text-gray-300 line-clamp-3`
   - 元信息：日期 + 分类标签（标签使用 `bg-primary/20 text-primary px-2 py-0.5 rounded-full text-xs`）
   - 底部：“阅读更多 →” 链接，颜色 `#3b82f6`，hover 下划线。

## 五、分页

- 使用“上一页 / 下一页”按钮，居中排列。
- 按钮样式：`border border-primary text-primary hover:bg-primary/10 px-4 py-2 rounded-lg transition`。

## 六、页脚

- 背景 `#030614`，上边框 `1px solid #1e293b`。
- 内容居中：导航链接（首页、归档、关于等）、社交链接（GitHub、公众号）、版权信息（`© 2026 黄鑫 (Frank) · 保持好奇，耐心打磨`）。
- 可选：备案号、友情链接。

## 七、响应式与交互

- 移动端汉堡菜单：点击时显示垂直菜单，再次点击或点击链接后关闭。
- 导航栏滚动时添加类 `scrolled`，修改背景和阴影。
- 所有 hover 过渡 `duration-200`。

## 八、注意事项

- 保持原有的数据获取逻辑（`asyncData` 或 `fetch`）不变。
- 不修改文章详情页和其他页面，仅重构首页。
- 确保 Tailwind CSS 已正确配置且 `content` 包含所有 Vue 文件。
- 测试环境：`npm run dev`，检查无控制台错误，各设备布局正常。

## 九、输出要求

请按照上述要求，生成完整的 `pages/index.vue`、`layouts/default.vue`、`components/AppHeader.vue`、`components/AppFooter.vue` 代码，以及必要的全局样式修改。确保 Logo 文字为“EasyBlog”，且整体高级感与作品集统一。
